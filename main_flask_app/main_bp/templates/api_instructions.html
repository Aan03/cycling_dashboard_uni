Cycling Parking API Instructions:

  • Where a username or password is required in the JSON request body, this refers to an existing user.
  • On this website, the "Download Data" page has been designed to use the API to generate a downloadable CSV file of reports for specified boroughs.
  • All the features on the website to do with report management (report creation, deletion and editing) can also be done through the API as long as the user details in the JSON body request are already in the system.
  • (Routes were tested using Postman.co)
API GET Routes:
  • http://127.0.0.1:5000/api/reports
    • Get all reports.
  • http://127.0.0.1:5000/api/reports/borough/[Enter borough name]
    • Get all reports for a specific borough.
    • Example: Enfield
  • http://127.0.0.1:5000/api/reports/rack/[Enter rack ID]
    • Get all reports for a specific bike rack.
    • Example: RWG148175

API POST Routes:
  • http://127.0.0.1:5000/api/reports/create
    • Create a new report.
    • JSON request body format for this request:
      {"username" : "[Enter username of report creator]",
      "password" : "[Enter password of report creator]",
      "rack_id" : "[Enter bike rack ID]",
      "details" : "[Enter report details]"}

API PUT Routes:
  • http://127.0.0.1:5000/api/reports/edit/[Enter report ID]
    • Edit an existing report.
    • JSON request body format for this request:
      {"username" : "[Enter username of report creator]",
      "password" : "[Enter password of report creator]",
      "details" : "[Enter report details]"}

API DELETE Routes:
  • http://127.0.0.1:5000/api/reports/delete/[Enter report ID]
    • Delete an existing report.
    • JSON request body format for this request:
      {"username" : "[Enter username of report creator]",
      "password" : "[Enter password of report creator]"}
